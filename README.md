Task 1: Многопоточная сортировка слиянием

Описание

Данный проект представляет собой реализацию алгоритма многопоточной сортировки (Parallel Merge Sort) на языке C с использованием библиотеки POSIX Threads (pthreads).

Программа предназначена для эффективной обработки массивов целых чисел путем распределения вычислительной нагрузки между несколькими потоками исполнения.

Функциональные возможности

Настройка количества потоков: пользователь может указать число потоков через аргументы командной строки.

Динамическое управление памятью: чтение данных из стандартного ввода (stdin) осуществляется с использованием динамического перераспределения памяти (realloc), что позволяет обрабатывать массивы произвольной длины.

Валидация входных данных: реализована проверка вводимых символов на соответствие целочисленному типу.

Параллельная обработка: использование стандарта C11 и библиотеки pthreads для ускорения процесса сортировки на многоядерных системах.

Стек технологий

Язык программирования: C (стандарт C11).

Библиотеки: pthreads (POSIX Threads).

Система сборки: GNU Make.

Компилятор: GCC.

Инструкция по сборке и запуску

Сборка проекта

Для компиляции исходного кода в исполняемый файл необходимо выполнить команду:

make

В результате сборки будет создан файл task1.

Запуск программы

Программа принимает один необязательный аргумент — количество потоков. По умолчанию используется 1 поток. Данные для сортировки подаются в стандартный поток ввода.

Пример запуска с указанием 4 потоков:

./task1 4

После запуска введите числа, разделяя их пробелами или переносами строки. Для завершения ввода нажмите Ctrl+D.

Пример использования с файлом через конвейер:

cat input.txt | ./task1 2

Очистка временных файлов

Для удаления объектных файлов и исполняемого файла выполните:

make clean

Технические подробности реализации

Распределение данных: Программа делит исходный массив на равные сегменты (чанки) для каждого потока. Если размер массива не кратен числу потоков, остаток распределяется между первыми потоками.

Этап сортировки: Каждый поток независимо сортирует свой сегмент, используя стандартную функцию qsort с пользовательской функцией сравнения.

Этап слияния: После завершения всех потоков основной поток производит попарное слияние отсортированных сегментов до получения финального упорядоченного массива.

Безопасность: В коде предусмотрены проверки на ошибки выделения памяти и корректность создания/завершения потоков.
